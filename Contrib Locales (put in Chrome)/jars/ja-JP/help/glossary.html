この用語集は、情報提供の目的のみにおいて提供され、以下に定義される用語、または解説されているテクノロジーのプライバシーやセキュリティの派生物として、完全で正統的な解説として信頼できるものではありません。

用語集

認証：  パスワード、証明書、個人識別番号（PIN）、その他の情報を使って、コンピュータ ネットワーク上で個人情報を確認すること。「パスワードベースの認証」、「証明書ベースの認証」、「クライアント認証」、「サーバー認証」の項も参照。

ブックマーク：  保存された Web ページアドレス（URL）。これにより、パーソナルツールバーにあるブックマークアイコンをクリックすることで、または［ブックマーク］メニューから該当するブックマークを選択することで、簡単に Web ページにアクセスできます。

CA：  認証局（CA）の項を参照。

認証局（CA）証明書：  認証局を識別する証明書。「認証局(CA)」、「従属認証局」、「ルート認証局」の項も参照。

キャッシュ：  あなたのコンピュータのハードディスク、またはあなたのコンピュータのランダムアクセスメモリ（RAM）に保存された Web ページのコピーの集まり。ブラウザは、あなたが Web をブラウズする際に、こうしたコピーを収集します。キャッシュが既にコピーを持っている、ある特定の Web ページの URL をクリックまたは入力すると、ブラウザは、その Web ページのオリジナルとキャッシュを比較します。もし両者に違いがなければ、ブラウザは、元のページを取り込まずにそのキャッシュを利用して、処理やダウンロードにかかる時間を節約します。

カレンダー：  あなたがスケジュールした、またはそれに関連するタスクを追跡するためのアプリケーション。あなたが仕事で Mozilla を使用している場合、このアプリケーションは Mozilla 内のウィンドウメニューから利用できる場合があります。あなたが家庭で Mozilla を使用している場合、またはあなたの仕事場で Mozilla カレンダーが利用できない場合、www.netscape.com （英文）で Web ベースのカレンダーを利用することができます。

証明書：  ID カードをデジタル化したもの。証明書では、個人、会社、またはその他の団体の名前を特定し、証明書に含まれている公開鍵がその団体に属していることを証明します。メッセージやその他のデータに対してデジタル署名を行うと、証明書に含まれている公開鍵に対応する秘密鍵を使って、そのメッセージに対するデジタル署名が作成されます。証明書は、認証局が発行し、デジタル署名を行います。証明書の有効性は、認証局のデジタル署名を確認することで検証できます。また、デジタル ID、デジタルパスポート、公開鍵証明書、X.509 証明書、およびセキュリティ証明書ともいいます。「公開鍵暗号方式」の項も参照。

認証局（CA）：  個人や、その証明書が識別する団体の ID を検証した後、証明書を発行するサービス。認証局はまた、証明書の更新や破棄、破棄された証明書リストの定期的な作成も行います。認証局は、独立したベンダー（クライアント証明書などに一覧表示された CA）の場合もあれば、証明書を発行するサーバーソフトウェア（Mozilla 証明書管理システムなど）を使用している個人や団体の場合もあります。「証明書」、「証明書破棄リスト（CRL）」の項も参照。

証明書のバックアップパスワード：  バックアップ中、またはバックアップ済みの証明書を保護するパスワード。証明書をバックアップしようとすると、証明書マネージャがパスワードを設定するよう指示します。また、バックアップ済みの証明書を復元しようとする際にも、パスワードの設定が要求されます。

証明書ベースの認証：  証明書と公開鍵暗号方式を基にして個人情報を確認すること。「パスワードベースの認証」の項も参照。

証明書チェーン：  連続する認証局によって署名された、一連の証明書の階層構造。認証局証明書は、認証局を識別するためのもので、その認証局によって発行された証明書に署名する際に使用されます。認証局証明書は同様に、親 CA の認証局証明書によって署名されます。このプロセスは、ルート認証局までさかのぼって行われます。

証明フィンガープリント：  証明書に関連付けられた固有の番号。この番号は証明書自体には含まれず、証明書の内容に関数を適用して生成されます。証明書の内容を一文字でも変更すると、関数によって別の番号が生成されます。このため、証明書フィンガープリントを使って、証明書が変更されていないことを検証することができます。

証明書マネージャ：  証明書を表示および管理できるブラウザの一部。証明書マネージャのメインウィンドウを表示するには、［編集］メニューを開き、［設定］を選択し、［プライバシーとセキュリティ］をクリックして、続いて［証明書マネージャ］をクリックします。

証明書の更新：  有効期限切れになる証明書を更新するプロセス。

証明書破棄リスト（CRL）：  認証局が作成および署名した証明書のうちで、すでに破棄された証明書のリスト。最新の CRL をブラウザまたはサーバーにダウンロードし、証明書が有効かどうか確認してから、その証明書を使用して認証設定を行うことができます。

証明書の検証：  証明書マネージャは、証明書を確認する際、その認証局独自の認証局証明書が、証明書マネージャにファイルされていること、およびその種の証明書の発行に関して、「信頼できる」認証局によってデジタル署名が作成されたことを確認します。また、確認中の証明書が、「信頼できない」とマークされていないかどうかも確認します。その後、OCSP（オンライン証明書ステータスプロトコル）が有効になっている場合、証明書マネージャはオンラインチェックも行います。証明書マネージャは、証明書自体またはブラウザの確認設定のいずれかで指定された URL に保存されている有効な証明書リストから証明書を検索します。これらのいずれかのチェックに失敗すると、証明書マネージャは証明書を「検証不可」とし、個人情報を認識しません。

暗号：  「暗号化のアルゴリズム」の項を参照。

クライアント：  サーバーに要求を送信したり、サーバーからの情報を受信したりする、ブラウザなどのソフトウェア。サーバーは通常、別のコンピュータ上で実行されています。クライアントソフトウェアを実行しているコンピュータも、クライアントといいます。

クライアント認証：  サーバーに対してクライアントを識別するプロセス。識別には例えば、名前とパスワード、またはクライアント SSL 証明書、およびデジタル署名されたデータなどが使用されます。「SSL（Secure Sockets Layer）」、「サーバー認証」の項も参照。

クライアント SSL 証明書：  SSL（Secure Sockets Layer）プロトコルを使って、ブラウザソフトウェアなどのクライアントが、クライアントの ID 情報（または、クライアントを使用するユーザーの個人情報）を認証設定するために、サーバーに提供する証明書。「クライアント認証設定」の項も参照。

コンポーネントバー：  Mozilla ウィンドウの左下にあるツールバー。コンポーネントバーを使って、Navigator、Mail & Newsgroups、Composer のためのアイコンをクリックして、Mozilla コンポーネントを切り替えることができます。

cookie：  Web サイトによって、あなたのコンピュータ上に保存される、小さな情報。cookie を利用するサイトに訪れると、そのサイトは、あなたのハードディスク上に一個または複数の cookie を設置するように、あなたのブラウザに要請します。その後、あなたがそのサイトに戻った時、あなたのブラウザは、そのサイトに属する cookie を送ります。cookie を使用することで Web サイトは、あなたのショッピングカートの内容といった、あなたについての情報を追跡することができます。あなたは、cookie がどのように利用され、どの程度の情報を Web サイトの cookie に保存するかを設定することができます。「外部 cookie」の項も参照。

Cookie マネージャ：  cookie を管理するために利用するブラウザの一部。詳細情報は、「Cookie マネージャを使う」の項を参照。

暗号化のアルゴリズム：  暗号化や復号化などの暗号化処理を実行する際に使用される規則や命令のセット。暗号化方式 ともいいます。

暗号：  情報をスクランブル（暗号化）およびアンスクランブル（復号化）する技術や手法。例えば、暗号化技術は、商業用 Web サイトとあなたのブラウザとの間を流れるスクランブルされていない情報を、スクランブルするのに利用されます。「公開鍵暗号」の項も参照。

復号化：  暗号化されたデータをアンスクランブルするプロセス。「暗号化」の項も参照。

デジタル ID：  「証明書」の項を参照。

デジタル署名：  署名されるデータと署名者の秘密鍵の両方から作成されるコード。このコードは、新規データごとに新しいものが作成され、同じものはありません。メッセージにコンマが一つ追加されても、そのメッセージのためのデジタル署名が変更されます。適切なソフトウェアによってデジタル署名が問題なく検証されると、ユーザーがトランザクションまたはメッセージを承認したことが証明されるだけでなく、デジタル署名された後でデータが変更されていないことも証明されます。デジタル署名は手書きの署名とは何の関係もありませんが、法律上、同様の目的で使用されることもあります。「否認防止」、「タンパー検出」の項も参照。

識別名（DN）：  証明書の対象を一意に識別する、特殊形式の名前。

デュアルキーペア：  二つの異なる証明書に対応する公開鍵と秘密鍵のペア二つで、合計四つの鍵。一方のペアの秘密鍵は署名処理に、もう一方のペアの公開鍵と秘密鍵は暗号化と復号化処理に使用されます。各ペアは、それぞれ個別の証明書に対応しています。「公開鍵暗号方式」の項も参照。

盗聴：  対象以外の団体や個人によって、ネットワーク上で送信された情報が不正に傍受されること。

暗号化：  情報の意味を第三者に理解できないようにスクランブルする処理。例えば、コンピュータ間での接続を暗号化すると、第三者はそこでやり取りされている情報をアンスクランブル、つまり復号化 することが困難になります。暗号化された情報は、適切な鍵を所有するユーザーだけが解読できます。「公開鍵暗号方式」の項も参照。

暗号化証明書：  暗号化のためだけに使用される秘密鍵に対応する公開鍵を持つ証明書。暗号化証明書は、署名処理には使用されません。「デュアルキーペア」、「署名証明書」の項も参照。

暗号化鍵：  暗号化のためだけに使用される秘密鍵。暗号化鍵とそれに対応する公開鍵、および署名鍵とそれに対応する公開鍵から、デュアルキーペアが構成されています。

XSLT（Extensible Stylesheet Language Transformation）：  XML ドキュメントを、別の XML ドキュメントまたは他の形式に変換する際に利用される言語。Mozilla の XSLT サポートについての情報は、「詳細設定 — 詳細」を参照してください。

XML（Extensible Markup Language）：  データ記述のオープン規格。HTML とは異なり、XML では、Web ページの開発者が特殊なタグを定義できます。詳細情報は、オンラインの W3C ドキュメント、Extensible Markup Language（XML）を参照してください。［訳注： 日本語での情報も参考にしてください。］

FTP（File Transfer Protocol）：  ユーザーが、ネットワークを介して、あるコンピュータから別のコンピュータにファイルを転送することができるようにする規格。ブラウザの FTP 機能を使って、ファイルを取って来ることができます。

フィンガープリント：  「証明フィンガープリント」の項を参照。

FIPS PUBS 140-1：  FIPS PUBS 140-1（Federal Information Processing Standards Publications）は、暗号化モジュール、つまり、データの暗号化と復号化やその他の暗号処理（デジタル署名の作成や確認など）を実行するハードウェアまたはソフトウェアを実装するための、アメリカ合衆国政府の規格です。アメリカ合衆国政府に対して販売される製品の多くは、FIPS 標準のいずれかの仕様に適合することが求められます。

外部 cookie：  あなたが訪れたサイトとは別のサイトによって保存される cookie。Web サイトは時として、別の Web サイトでホストされている内容を表示することがあります。そうした内容は、画像にはじまりテキストや広告にまで至ります。そのような要素をホストする Web サイトはまた、たとえあなたがそのサイトを直接訪れなくても、あなたのブラウザに cookie を保存する機能も備えています。「第三者 cookie」としても知られます。

フォームマネージャ：  あなたの名前、住所、電話番号といった、あなたがオンラインフォームに入力した個人情報を保存する補助をするブラウザの一部。Web サイトがフォームを提示した場合、フォームマネージャは、自動的にそのフォームを入力することができます。詳細は、「フォームマネージャを使う」を参照してください。

ヘルパーアプリケーション：  ブラウザによってダウンロードされたファイルを開いたり、表示するために利用される、あらゆるアプリケーション。プラグインとは、特別な種類のヘルパーアプリケーションのことで、メインとなるブラウザがインストールされたディレクトリにある［Plugins］ディレクトリにインストールされ、通常ブラウザそのものによって（内部で）開かれます。Microsoft Word、Adobe Photoshop その他の外部アプリケーションは、プラグインではなく、ヘルパーアプリケーションとして捉えられます。なぜなら、そうしたアプリケーションは、ブラウザディレクトリにインストールされず、ダウンロードダイアログボックスから開かれるからです。

ホームページ：  ブラウザを立ち上げた時や、［ホーム］ボタンをクリックした時に表示されるページ。また、サイト全体を閲覧できる、Web サイトのメインページを指す場合にも利用されます。

HTML（Hypertext Markup Language）：  Web ページのために利用されているドキュメント形式。HTML 標準は、タグまたはコードを定義し、Web ページを構成する、テキストレイアウト、フォント、スタイル、画像その他の要素を定義するために使用されます。

黙諾：  黙示の同意、または「オプトアウト」承諾としても知られます。Web サイトが目的を明らかにしたページで、あなたがオプションを選択して明示的に拒否しない限り、Web サイトがあなたに関する情報を（cookie やオンラインフォームといったかたちで）収集することを許可する、プライバシー設定を説明するために利用される用語。実際に情報が収集される時にも、あなたの承諾は求められない可能性があります。

IMAP（Internet Message Access Protocol）：  あなたの全てのメッセージやメッセージへの変更を、あなたのコンピュータのハードディスク上にではなく、サーバー上に保存することができるようにする、標準的なメールサーバープロトコル。POP ではなく IMAP を利用することで、ディスクスペースの節約に役立ち、送信済みメール、下書き、そしてカスタムフォルダーを含めた全てのメールボックスに、どこからでもアクセスできます。モデムを通して IMAP を使用すると、一般的に POP メールサーバーよりも接続が早くなります。というのも、はじめの段階ではメッセージヘッダーのみをダウンロードするからです。全てのプロバイダが IMAP に対応しているわけではありません。

インターネット：  TCP／IP といった規格化されたプロトコルを利用して、多数のコンピュータがお互いに通信し合う、世界規模のネットワーク。もともとは、1969年にアメリカ軍のために開発され、教育・研究機関を含んで発展し、1990年代後半、数多くの企業、団体および個人が参加するようになりました。今日では、インターネットは、電子メール、World Wide Web の閲覧、インスタントメッセージ、ユーズグループ、その他の多数の目的に利用されています。

IP アドレス（Internet protocol address）：  TCP／IP ネットワーク上のコンピュータのアドレス。インターネット上の全てのコンピュータは、IP アドレスを持っています。クライアントは、固定 IP アドレス、またはネットワークに接続する度に動的に割り振られる IP アドレスのどちらかを持っています。IP アドレスは、204.171.64.2 といったように、四つの数字の組み合わせで表現されます。

Java：  Sun Microsystems によって開発されたプログラミング言語。一つの Java プログラムは、多数の異なった種類のコンピュータ上で動かすことができ、そのため、プログラマーはそれぞれの種類のコンピュータのために個別のバージョンを作成する必要がなくなります。あなたのブラウザは、Java プログラム（アプレットと呼ばれます）を自動的にダウロードし、動かすことができます。

JavaScript：  Web ページを構成するために一般的に用いられるスクリプト言語。プログラマーは、JavaScript を利用して、例えば、フォームやボタンを表示するなど、Web ページをよりインタラクティブにできます。JavaScript は Java と共に利用できますが、技術的には異なる言語です。Java は、JavaScript がなくても正常に作動します。

鍵：  データを暗号化または復号化するために暗号化のアルゴリズムで使用される非常に大きい数値。例えば、あるユーザーの公開鍵を使用して、そのユーザーに送信したメッセージを他のユーザーが暗号化することができます。暗号化したメッセージを解読するには、対応する秘密鍵を使用しなければなりません。「公開鍵暗号方式」の項も参照。

LDAP（Lightweight Directory Access Protocol）：  複数のプラットフォームで、企業のアドレス帳といった、ディレクトリサービスにアクセスするための規格化されたプロトコル。あなたのブラウザが LDAP ディレクトリにアクセスできるようにするには、アドレス帳から設定できます。また、電子メールアドレスの自動補完のために LDAP ディレクトリを利用できるようメール & ニュースグループを設定することもできます。

ロケーションバー：  URL や検索したい言葉を入力できる、Navigator ウィンドウの上部にあるフィールド（と関連するボタン）。詳細は、「別のページに移る」を参照してください。

マスターキー：  証明書マネージャが情報の暗号化に使用する対称鍵。たとえば、パスワード マネージャは証明書マネージャとマスターキーを使用し、電子メールのパスワード、Web サイトのパスワードなど、保存されている機密情報を暗号化します。「対称暗号化」の項も参照。

マスターパスワード：  セキュリティーデバイスに格納されているマスターキーや秘密鍵を保護するために、証明書マネージャが使用するパスワード。証明書マネージャは、例えば、ユーザーが電子メールメッセージに署名したり、Web サイトに対して本人であることを証明したりするために、ユーザー独自の証明書を使用する場合、ユーザーの秘密鍵にアクセスする必要があります。 パスワードマネージャまたはフォームマネージャが、個人情報の読み取りや追加を行う場合、証明書マネージャはユーザーのマスターキーにアクセスする必要があります。 マスターパスワードの設定や変更は、［マスター パスワード］設定パネルで行います。セキュリティーデバイスごとに異なるマスターパスワードを設定してください。「秘密鍵」、「マスターキー」の項も参照。

不当表示：  ユーザーまたは団体を実際とは異なるものとして表示すること。例えば、ある Web サイトを家具店のサイトのように表示し、実際にはクレジットカードによる支払いを行わせるだけで、商品を配送しない場合などがあります。「なりすまし」の項も参照。

Navigation ツールバー：  ［戻る］や［進む］ボタンを含む、ブラウザウィンドウの上部にあるツールバー。

Netscape Quality Feedback Agent：  プログラムが予期しない終了をした時に、技術情報を収集する Mozilla に組み込まれた小さなソフトウェア。あなたの許可を尋ねた後、そのソフトウェアは、収集した情報を mozilla.org に送ります。参加するという選択をした場合、クラッシュが起きた時に何が起きていたかコメントを加えることができます。mozilla.org はそうしたフィードバックを利用して、バグをフィックスしパフォーマンスを向上させます。このソフトウェアや関連するサービスは、「talkbalk」と呼ばれることもあります。詳細情報は、オンラインドキュメントの Netscape Quality Feedback System for Netscape （英文： リンク切れ）を参照してください。

否認防止：  メッセージ送信者が、メッセージを送信したことを否定できないようにすること。通常の手書きによる署名は、否認防止形式の一つです。デジタル署名もその一つです。

オブジェクト署名：  ソフトウェア開発者が Java コード、JavaScript スクリプト、その他各種ファイルに署名できるようにしたり、ユーザーが送信元（署名者）を識別し、署名されたコードを使ったローカルシステムリソースへのアクセスを制御する技術。

オブジェクト署名証明書：  コードファイルなどのオブジェクトへの署名に使用される、対応の秘密鍵を持つ証明書。「オブジェクト署名」の項も参照。

OCSP（オンライン証明書ステータスプロトコル）：  証明書を使用するたびに、電子メール証明書の妥当性をオンラインで確認するために証明書マネージャが使用する規則セット。このプロセスには、特定の Web サイトで保存された有効な証明書リストに対して、その証明書を確認します。OCSP が有効に動作するためには、コンピュータがオンライン状態でなければなりません。

パスワードベースの認証：  名前とパスワードを使用した信頼性の高い識別方法。「認証」の項も参照。

パスワードマネージャ：  あなたのコンピュータのハードディスクに、あなたのユーザー名やパスワードを一部または全て保存して、そのような情報が必要なサイトをあなたが訪れた際には自動的に入力してくれる、ブラウザの一部。詳細は、「パスワードマネージャを使う」の項を参照してください。

パーソナルツールバー：  Navigator で標準のロケーションバーの下に表示される、カスタマイズ可能なツールバー。このツールバーには、［ホーム］、［検索］、［ブックマーク］などの、追加／削除可能なボタンがついています。また、お気に入りのブックマークや、ブックマークを含むフォルダのためのボタンを追加することも可能です。詳細は、「パーソナルツールバー」の項を参照してください。

PKCS #11：  スマートカードなどのセキュリティデバイスを制御する公開鍵暗号方式規格。「セキュリティーデバイス」、「スマートカード」の項も参照。

PKCS #11 モジュール：  PKCS #11 規格を使った暗号化と復号化といった、暗号化サービスを管理するあなたのコンピュータ上のプログラム。暗号モジュール、暗号サービスプロバイダ、またはセキュリティモジュール ともいいます。PKCS #11 モジュールは、ハードウェアまたはソフトウェアのいずれかのデバイスを管理します。PKCS #11 モジュールは、常に一つまたは複数のスロットを管理します。スロットは、（例えば、スマートカードの読み込みなどの）物理的なリーダーの一種として、またはソフトウェアとして実装できます。PKCS #11 モジュール用の各スロットには、同様にセキュリティーデバイス（トークン ともいいます）を格納することもできます。セキュリティーデバイスは、暗号化サービスを提供し、証明書と鍵を保存できるハードウェア、またはソフトウェアデバイスです。証明書マネージャには、二つの PKCS#11 モジュールがあります。スマートカードリーダーやその他のハードウェアデバイスを管理するために、あなたのコンピュータに追加モジュールをインストールすることもできます。

P3P（Platform for Privacy Preferences）：  ユーザーが訪れた Web サイトによって個人情報が利用されるのを、ユーザー側に管理権を与えようとデザインされ、W3C（World Wide Web Consortium）によって公開された規格。この規格をサポートした Web サイトで cookie を利用するにあたっての情報は、「プライバシーレベルの設定」の項を参照してください。この規格そのものについての一般的な情報は、オンラインドキュメント、P3P Public Overview （英文）を参照してください。

プラグイン：  オーディオやビデオクリップを再生する機能といった、ブラウザに新しい機能を追加するヘルパーアプリケーションの一種。他のタイプのヘルパーアプリケーションとは異なり、プラグインアプリケーションは、自身をブラウザのメインインストレーションディレクトリにインストールし、通常ブラウザそのものの中（内部）で開かれます。例えば、オーディオプラグインでは、Web ページや電子メールに含まれたオーディオファイルを聴けるようにします。Macromedia Flash Player や Java は、どちらもプラグインアプリケーションの例として挙げられます。

POP（Post Office Protocol）：  あなたのローカルコンピュータに新しいメッセージをダウンロードするよう要求する（だたし、サーバー上にメッセージのコピーを残すこともできます）、標準のメールサーバープロトコル。POP を利用すると、送信済みメール、下書き、そしてカスタムフォルダーを含む全てのメッセージを、そのコンピュータのみに保存することができます。これとは対照的に、IMAP では、あらゆるコンピュータからアクセスできるサーバー上に、全てのメッセージや変更を永続的に保存することができます。ほとんどのプロバイダが POP をサポートしています。

秘密鍵：  公開鍵暗号方式で使用されるペアの鍵の一つ。秘密鍵は他人に公開されずに管理され、対応する公開鍵で暗号化されたデータを解読する際に使用されます。

プロキシ：  他のクライアントに代わって要求をする目的で、サーバーとクライアントの両方の役割を果たす、中継または「橋渡し役」プログラム。

公開鍵：  公開鍵暗号方式で使用されるペアの鍵の一つ。公開鍵は、証明書の一部として自由に配布および公開されます。一般に、公開鍵は公開鍵の所有者に送られるデータを暗号化するために使用されます。その後、データを受け取った公開鍵の所有者は、対応する秘密鍵を使ってデータを復号します。

公開鍵暗号方式：  個人、組織、またはルータのようなハードウェアなどのエンティティが、その ID 情報を電子的に確認したり、電子データに署名して暗号化したりするのに幅広く使用される技術とその規格セット。公開鍵暗号方式には、公開鍵と秘密鍵という二つの鍵があります。公開鍵は、証明書の一部として公開されます。その証明書は、その公開鍵を特定の個人情報に関連付けます。対応する秘密鍵は他人に公開されずに管理されます。公開鍵で暗号化されたデータは、対応する秘密鍵でしか解読できません。

PKI（公開鍵インフラストラクチャ）：  ネットワーク環境での公開鍵暗号方式と証明書の使用をサポートする規格とサービス。

ルート認証局：  証明書チェーンの階層の一番上にある自己署名付き証明書を持つ認証局。「従属認証局」の項も参照。

検索エンジン：  ユーザーが World Wide Web から特定の情報を検索したり、取り込むことができるようにする Web ベースのプログラム。検索エンジンは、Web ドキュメントの全文検索、キーワードの一覧検索、または Web ドキュメントを調べ、人の手で索引化する司書を利用した検索を行います。通常、ユーザーが入力欄に単語や熟語（照会ともよばれます）を入力すると、検索エンジンは、関連する Web ページのリンクを表示します。

SSL（Secure Sockets Layer）：  認証設定と暗号化された接続の確立を目的とし、クライアントとサーバー間の相互認証設定を可能にするプロトコル。SSL の下位層は TCP/IP 上に構築され、HTTP、LDAP、IMAP、NNTP、およびその他のハイレベルのネットワークプロトコルより下位で実行されます。TLS（Transport Layer Security）と呼ばれる新しい ITEF（Internet Engineering Task Force）規格は、SSL を基に作られています。「認証」、「暗号化」の項も参照。

セキュリティ証明書：  「証明書」の項を参照。

セキュリティデバイス：  暗号化と復号化などの暗号サービスを提供し、証明書と鍵を保存するハードウェアやソフトウェア。ハードウェアに実装されるセキュリティ デバイスの一つにスマートカードがあります。 証明書マネージャには、ブラウザを実行するときに常に有効になる、ソフトウェアセキュリティデバイスと呼ばれるセキュリティデバイスが備わっています。各セキュリティデバイスは、専用のマスターパスワードによって保護されています。

セキュリティモジュール：  「PKCS #11 モジュール」の項を参照。

セキュリティトークン：  「セキュリティデバイス」の項を参照。

サーバー：  Web ページを提供するソフトウェアなど、クライアントから要求を受信し、情報をクライアントに送信するソフトウェア。クライアントは通常、別のコンピュータ上で実行されています。サーバー ソフトウェアが実行しているコンピュータもサーバーといいます。

サーバー認証：  サーバー SSL 証明書を使用して、クライアントに対するサーバーを識別するプロセス。「クライアント認証設定」、「SSL（Secure Sockets Layer）」の項も参照。

サーバー SSL 証明書：  SSL （Secure Sockets Layer）プロトコルを使用してサーバーの ID 情報を認証するために、サーバーがクライアントに提供する証明書。

署名証明書：  受信者が送信者の個人情報を確認できるように、転送データの署名に使用される対応秘密鍵に対応する証明書。認証局は、多くの場合、電子メールメッセージの暗号化に使用する暗号化証明書の発行と同時に、電子メールメッセージの署名に使用する署名証明書を発行します。「デュアルキーペア」、「デジタル署名」の項も参照。

署名鍵：  署名のためだけに使用される秘密鍵。デュアルキーペアは、署名鍵に対応する公開鍵と、暗号化鍵に対応する公開鍵から構成されています。

スロット：  ハードウェアの一部、またはソフトウェア内でそれに相当する部分を指します。スロットは、PKCS #11 モジュールによって制御され、セキュリティデバイスを格納できるように設計されています。

スマートカード：  通常、マイクロプロセッサを搭載したクレジットカードサイズの小型デバイス。［訳注： 日本では IC カードと呼ばれます。］スマートカードにはマイクロプロセッサが備わっており、（鍵や証明書などの）暗号情報が保存され、暗号処理を実行する機能が備わっています。スマートカードは、PKCS #11 規格を使用しており、セキュリティデバイスの一種です。

ソフトウェアセキュリティデバイス：  ユーザーの証明書に関連付けられた秘密鍵を保存するために証明書マネージャが使用する、標準設定のセキュリティデバイス。ソフトウェアセキュリティデバイスには、秘密鍵の他に電子メールパスワード、Web サイトパスワード、その他の機密情報を暗号化するためにパスワードマネージャが使用する、マスターキーも保存されます。「秘密鍵」、「マスターキー」の項も参照。

なりすまし：  別の誰かのふりをすること。たとえば、ユーザーが実際には存在しない jdoe@mozilla.com という電子メール アドレスを持つユーザーになったふりをしたり、コンピュータが実際には存在しない www.mozilla.com というサイトのふりをすることです。なりすましは、不当表示の一種です。

SSL：  「SSL （Secure Sockets Layer）」の項を参照。

ステータスバー：  Mozilla ウィンドウの下部に表示されるツールバー。ステータスバーには、左側にコンポーネントバー、右側にステータスアイコンを表示します。例えば Navigator では、錠アイコンといったステータスアイコンを表示します。

サブジェクト：  証明書が識別する個人、組織、またはルータなどのエンティティ。特に、証明書のサブジェクトフィールドには、証明される対象となるサブジェクト名やその他の特性が保存されます。

サブジェクト名：  証明書のサブジェクトを記述する識別名（DN）。サブジェクトごとに異なる名前を使用します。

従属認証局：  別の従属認証局またはルート認証局によって署名される証明書を持つ認証局。「証明書チェーン」、「ルート認証局」の項も参照。

対称暗号化：  提供されたメッセージの暗号化と復号化とに一つの暗号鍵を使用する暗号化方式。

talkback：  「Netscape Quality Feedback Agent」の項を参照。

タンパー検出：  電子形式で受信したデータが勝手に変更されていないこと、つまり、受信したデータが元のデータと完全に一致していることを確認する仕組み。

TCP：  「Transmission Control Protocol／Internet Protocol（TCP／IP）」の項を参照。.

第三者 cookie：  「外部 cookie」の項を参照。

TLS：  「SSL（Secure Sockets Layer）の項を参照。

トークン：  「セキュリティデバイス」の項を参照。

TCP／IP（Transmission Control Protocol／Internet Protocol）：  様々なオペレーティングシステムを走らせているコンピュータを接続するために利用される Unix プロトコル。TCP／IP は、主要なインターネットプロトコルであり、グローバル規格となっています。

信頼：  個人ユーザーや他のエンティティを完全に信用すること。公開鍵インフラストラクチャ（PKI）において、通常、信頼とは証明書のユーザーと証明書を発行した認証局（CA）との関係を表します。証明書マネージャを使って、ある認証局を信頼するよう指定すると、証明書マネージャは、証明書ごとに設定を変更しない限り、その認証局が発行した有効な証明書をすべて信頼します。証明書マネージャの［認証局］タブを使って、特定の認証局が発行した証明書のうち、信頼できる／できない証明書の条件を指定できます。

URL（Uniform Resource Locator）：  あなたのブラウザに、例えば、http://www.netscape.com. のように、Web 上のファイルや他の資源をどのように見つけるかを伝える、規格化されたアドレス。ブラウザのロケーションバーに URL を入力して、Web ページにアクセスすることができます。URL はまた、クリックすると他の Web ページに行くことができるような Web ページ上のリンクとしても利用されます。インターネットアドレス、または Web アドレスとしても知られます。

Web ページ：  固有のアドレスまたは URL により指定され、テキスト、ハイパーリンクまたは画像を含む、World Wide Web 上のドキュメント。

Web サイト：  ハイパーリンクによりリンクされ、単一の企業、団体、または個人により管理される、関連した Web ページの一群。Web サイトには、テキスト、画像、オーディオそしてビデオファイルや、他の Web サイトへのリンクが含まれます。

World Wide Web：  Web としても知られます。Web サーバーにより保存され、（Mozilla といった）Web ブラウザと呼ばれるクライアントにより表示される、Web ページからなるインターネットの一部。
